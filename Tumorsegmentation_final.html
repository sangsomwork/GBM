Automated Glioblastoma Tumor Segmentation¶

MRI T1 and FLAIR images enable visualization of tumor segmentation. The code below utilizes the provided FLAIR and T1 image to perform automated tumor segmentation and generate the following label files:

  • Total tumor (from FLAIR image).
  • Tumor bulk (from T1), which consists of the contrast enhancing (CE) and necrotic (NE) parts of the tumor.

Additionally, it computes the dice coefficient of overlap between the tumor bulk and the provided label file to evaluate accuracy

Methodology¶

The following steps are executed to obtain the segmentation:

  • Registration of FLAIR to T1 image
  • Skull stripping of T1 and FLAIR images using atlas based segmentation
  • Total tumor segmentation from FLAIR
    • Automated thresholding for initial segmentation of hyperintense region
    • Refinement using shape and size constraints to segment tumor
  • Segmentation of tumor bulk from T1
    • Thresholding to obtain CE region
    • NE region: Level set segmentation using eroded total tumor mask from FLAIR as initialization
  • Computation of dice coefficient of overlap

In [1] a semi-automated method using FSLView was used to obtain the total tumor and tumor bulk segmentation. Fully automated tumor segmentation approaches in the recent past rely on machine learning techniques such as in [2], where voxel-wise classification was performed. Since I was provided one image, I introduced shape and size constraints to segment the tumor.

Register FLAIR to T1¶

  • Metric: Mutual information is used since it is robust to intensity differences in the images.
  • Transformation: Rigid transformation (translation and rotation) since the images are of the same subject
  • Multiresolution framework (3 levels) for a fast, robust registration.
  • Initialization: The registration is initialized by finding the transform that superimposing the centers of the two images using CenteredTransformInitializer from SimpleITK.
  • Optimizer: Gradient descent
  • Interpolation: Linear The parameters were chosen to give a good visual overlap for this image.

Skull stripping through atlas based registration¶

I used the ICBM atlas http://nist.mni.mcgill.ca/?p=904 that includes a T1 image along with a brain mask. I define a skull strip function which allows the user to input any atlas+label file to perform the skull stripping. This function performs affine registration to account for differences in size of brain. I chose mutual information since it is robust to intensity differences between the fixed and moving images. The rest of the metrics are similar to the FLAIR/T1 registration.

In [38]:
# function for skullstripping
def skullstrip(image, atlas, label):
    
    fixed_image = image
    moving_image = atlas
    initial_transform = sitk.CenteredTransformInitializer(fixed_image, moving_image, 
                                                        sitk.AffineTransform(fixed_image.GetDimension()), 
                                                        sitk.CenteredTransformInitializerFilter.GEOMETRY)
    registration_method = sitk.ImageRegistrationMethod()

    # Similarity metric settings.
    registration_method.SetMetricAsMattesMutualInformation(numberOfHistogramBins=50)
    registration_method.SetMetricSamplingStrategy(registration_method.RANDOM)
    registration_method.SetMetricSamplingPercentage(0.1)

    registration_method.SetInterpolator(sitk.sitkLinear)

    # Optimizer settings.
    registration_method.SetOptimizerAsGradientDescent(learningRate=1.0, numberOfIterations=100, 
                                                      convergenceMinimumValue=1e-6, convergenceWindowSize=10)
    registration_method.SetOptimizerScalesFromPhysicalShift()

    # Setup for the multi-resolution framework.            
    registration_method.SetShrinkFactorsPerLevel(shrinkFactors = [4,2,1])
    registration_method.SetSmoothingSigmasPerLevel(smoothingSigmas=[4,2,0])
    registration_method.SmoothingSigmasAreSpecifiedInPhysicalUnitsOn()

    registration_method.SetInitialTransform(initial_transform, inPlace=False)

    final_transform = registration_method.Execute(sitk.Cast(fixed_image, sitk.sitkFloat32), 
                                                  sitk.Cast(moving_image, sitk.sitkFloat32))


    brainmask = sitk.Resample(label, fixed_image, final_transform, sitk.sitkNearestNeighbor,0.0,moving_image.GetPixelID())
    
    msk = sitk.MaskImageFilter()
    brainimage=msk.Execute(image,brainmask>0 )
    
    
    return brainimage, brainmask

Refine FLAIR tumor segmentation¶

The high intensity regions include the tumor, but also have some CSF. Additionally, in some cases, skull stripping might leave parts of skull. The following code isolates the tumor region by making the following assumptions:

  • Tumor region is more spherical than CSF or skull regions that tend to be elongated:
    • Ratio of equivalent ellipsoid diameters close to 1 indicate more spherical than elongated. I defined the Elong metric as the squared difference of ratio from 1. Values close to zero indicate more spherical.
  • We extract the largest hyperintense tumor in the brain, assuming we are only interested in the primary volume.

# generate label image
conncomp = sitk.ConnectedComponentImageFilter()
seg = conncomp.Execute(segFLAIR)

# find label characteristics 

stats = sitk.LabelShapeStatisticsImageFilter()
stats.Execute(seg)

labels = stats.GetLabels()
sizes=[]
elong = []
for lab in labels:
    dia=stats.GetEquivalentEllipsoidDiameter(lab)
    ratiodia = np.square(1- dia[1]/(dia[0]+1e-10))
    elong.append(ratiodia) 
    sizes.append(stats.GetPhysicalSize(lab))
sizes=np.asarray(sizes)
elong = np.asarray(elong)
labels = np.asarray(labels)

labels= labels[~np.isnan(elong)]
sizes = sizes[~np.isnan(elong)]
elong = elong[~np.isnan(elong)]

We assume the following to separate the tumor region from other parts that have high intensity (e.g., remnant skull pieces or CSF)

  • Elong metric < 0.25 ( 0 is perfect sphere)
  • Largest region (assuming we want to segment the largest tumor)

Segmentation of Necrotic region using Level Sets¶

I used the Threshold Segmentation Level Set filter, which uses level sets to propagate the front until it reaches the limits of an intensity window.

Intialization: The level set was initialized by eroding the total tumor label obtained from FLAIR.

In [105]:
binerode = sitk.BinaryErodeImageFilter()
binerode.SetKernelRadius(9) 
erodedtumormask = binerode.Execute(segFLAIR>0)
disp.myshow(sitk.LabelOverlay(img_T1brain_255[:,:,80], erodedtumormask[:,:,80]), "Level set initialization ")

To generate a feature image for the level set filter, we compute the Euclidean distance fuction of the binary FLAIR segmentation. The parameters of the level set were chosen as:

  • Intensity window: Lower threshold was chosen same as the one used above for CE segmentation. Upper threshold was chosen to be 3 times the lower threshold. This will lock the level set at a value greater than the CE threshold.
  • Propagation scaling: A positive value corresponds to expansion. Since we are expanding to fill the necrotic region, I chose a positive value.
  • Curvature scaling: Determines the smoothness of the resulting mask.
    The parameters were chosen below to give the best results visually. For a larger dataset, a grid search may be performed to obtain the best agreement (coefficient of overlap) with ground truth labels.
